## Momentum notes carried forward

1. **New this session:** when forcing a real container replacement in a test/demo (not just a config reload), don't rely on `docker compose up --force-recreate` alone — Docker's bridge-network IPAM can hand the recreated container its OLD ip back if it's the only one freed. Force a genuine IP difference via a second container sharing the same `--network-alias`, verified with `docker inspect`, before removing the original.
2. **New this session:** a service that fails by refusing/timing out a TCP connect (Nginx proxying to a dead IP, or similar) does not have deterministic failure timing unless a connect timeout is explicitly configured — treat multiple plausible failure status codes (502 AND 504) as the same real bug, not a sign something's flaky, unless a timeout is set to force one deterministic outcome.
3. Momentum notes from 2026-07-21/22 (proactive port-0/PID-scoping/matchAll-timeout discipline, `npm view` before writing any new dependency version, `@types/node` + `types: ["node"]` for TS examples referencing Node builtins, `error.stdout` for expected non-zero-exit CLI errors, `demo.js` staying a thin call-and-print layer, disabling a spawned CLI tool's own internal multiprocess/fork option from the start) all still stand — no new instances of any of those specific bugs recurred this session.
